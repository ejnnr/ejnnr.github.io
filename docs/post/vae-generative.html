VAEs from a generative perspective

January 6, 2021
Deep learning

This is my attempt to tell a story1 about how you might invent variational autoencoders (VAEs). There are already great introductory posts doing this and if you haven’t seen VAEs before, I would strongly recommend you start with one of those. These introductions often start with autoencoders and then extend them to VAEs. In contrast, we will start by asking ourselves how to generate new data that matches a training distribution and then motivate VAEs from there. We won’t assume an autoencoder-like architecture a priori, instead it will arise naturally from this motivation.

Of course just this motivation of generating new samples given a training distribution won’t uniquely lead to VAEs — after all, there are other good options for generative models. So at some points we will need to make design decisions but hopefully they won’t come out of the blue.

One pedagogical note before we start: if this derivation of VAEs seems unnecessarily long and convoluted, that’s because it is. The goal is not to arrive at the VAE framework as quickly as possible, but rather to make each step seem natural and to avoid any unmotivated “magical” jumps. It’s probably best if you forget for a moment what you know about VAEs, in particular that they consist of an encoder and a decoder. We will get there at the very end but initially this preconception might just be confusing.

Generative models

The goal in generative modeling is the following: we have some family of probability distributions P\mathcal{P}P. Given a set of training examples D\mathcal{D}D (assumed to be i.i.d.), we now want to pick the distribution p∈Pp \in \mathcal{P}p∈P from our family that maximizes the likelihood ∏x∈Dp(x)\prod_{x \in \mathcal{D}} p(x)∏x∈D​p(x). Equivalently, we can maximize the log-likelihood:

arg max⁡p∈P∑x∈Dlog⁡p(x)\argmax_{p \in \mathcal{P}} \sum_{x \in \mathcal{D}} \log p(x)p∈Pargmax​x∈D∑​logp(x)

For now, we will consider the simper special case where we only have a single datapoint xxx and want to maximize log⁡p(x)\log p(x)logp(x) (we will get back to the general case at the end).

Optimizing over a family of probability distributions is very abstract. To turn this into a problem we can actually solve numerically, we will use a parameterized family pθ(x)p_\theta(x)pθ​(x) and optimize over the parameter θ∈Rp\theta \in \R^pθ∈Rp. pθ(x)p_\theta(x)pθ​(x) should be differentiable with respect to θ\thetaθ, then we can at least find a local optimum for our problem using gradient ascent.

This still leaves the question which parameterized family we should use. This is the largest crossroads we’ll face in this post: there are many good options to choose from. The challenge we face is to find a good trade-off between having a flexible family of distributions and keeping the number of parameters manageable. For example, if xxx takes on discrete values, we could in principle use the full categorical distribution over all possible values of xxx. This would be as flexible as possible but the number of parameters might be huge. If xxx describes a 28×2828\times 2828×28 binary image, there are already 228⋅28=2784≈102362^{28 \cdot 28} = 2^{784} \approx 10^{236}228⋅28=2784≈10236 possible values that xxx can take, meaning we’d need about that many parameters.

The way we will deal with this problem is to use a continuous mixture of simple distributions. We will introduce a new latent variable z∈Rkz \in \mathbb{R}^kz∈Rk on which we define a very simple distribution p(z)p(z)p(z), for example a unit normal, z∼N(0;I)z \sim \mathcal{N}(0; I)z∼N(0;I). Then we parameterize a distribution pθ(x∣z)p_\theta(x|z)pθ​(x∣z), which gives us

pθ(x)=∫p(z)pθ(x∣z)dzp_\theta(x) = \int p(z) p_\theta(x|z) dzpθ​(x)=∫p(z)pθ​(x∣z)dz

The important point is that for a fixed zzz, pθ(⋅∣z)p_\theta(\cdot|z)pθ​(⋅∣z) may be an extremely simple distribution. In the example above, we could use an independent Bernoulli for each of the 784 pixels, which requires only 784 parameters. But because we additionally have a dependency on zzz, the marginal distribution pθp_\thetapθ​ can be much more complex (in particular, the pixels are typically not independent). Of course the dependency on zzz will require some additional parameters but this could just be a reasonably sized neural network, which gives us far fewer parameters than the 27842^{784}2784 that a full categorical distribution would require.

This already describes our model. Sampling from this model is easy: we sample z∼p(z)z \sim p(z)z∼p(z) and then for this zzz sample x∼pθ(x∣z)x \sim p_\theta(x|z)x∼pθ​(x∣z). By assumption, both of those distributions are very simple (and we can also choose them to be easy to sample from).

But evaluating the likelihood pθ(x)p_\theta(x)pθ​(x) of a datapoint is intractable for most models pθ(x∣z)p_\theta(x|z)pθ​(x∣z) and p(z)p(z)p(z) because it requires calculating a complicated integral. Even if we only care about generating samples, this is a problem: to train the model, we want to maximize log⁡pθ(x)\log p_\theta(x)logpθ​(x), but we can’t even evaluate it (nor its gradient, for the same reason).

The cleverness of VAEs lies in using the right approximations to make this optimization problem tractable, and that is what the remainder of this post is about.

Variational inference

First, we expand the log-likelihood a bit. For any value of zzz, we have

log⁡p(x)=log⁡p(z)p(x∣z)−log⁡p(z∣x)\log p(x) = \log p(z) p(x|z) - \log p(z|x)logp(x)=logp(z)p(x∣z)−logp(z∣x)

(not writing our the dependency on θ\thetaθ for now). The first term is easy to evaluate. So if we could evaluate the second term, our problem would be solved (sidenote on motivation2).

This is where variational inference comes into play (here is a tutorial if you want to dive a bit deeper but that’s not necessary for this post). The idea of variational inference is that you have some distribution ppp that you care about, but which is intractable to work with. So you define a family Q\mathcal{Q}Q of simpler distributions and then find

q∗:=arg min⁡q∈QD(q∥p)q^* := \argmin_{q \in \mathcal{Q}} D(q\Vert p)q∗:=q∈Qargmin​D(q∥p)

where D(⋅∥⋅)D(\cdot \Vert \cdot)D(⋅∥⋅) is the Kullback-Leibler divergence (which measures “distances” between probability distributions, though it is not a metric in the mathematical sense3). We can then use q∗q^*q∗ in place of ppp whenever we need to evaluate it.

This may sound like an enourmous amount of computational overhead: to just evaluate our objective, we will have to solve an entire optimization problem each time! We will later find a way to alleviate this issue but for now, let’s just ignore it and understand how we would solve the problem naively.

To apply this to our problem, we will approximate p(z∣x)p(z|x)p(z∣x) with a simpler distribution qλ(z)q_\lambda(z)qλ​(z), parameterized by a new parameter λ\lambdaλ. For example, qλq_\lambdaqλ​ could be a Gaussian and λ\lambdaλ would be its mean and covariance matrix. Note that while qλ(z)q_\lambda(z)qλ​(z) does not explicitly depend on xxx, the optimal parameter λ∗\lambda^*λ∗ does depend on xxx because we minimize the Kullback-Leibler divergence between qλq_\lambdaqλ​ and p(⋅∣x)p(\cdot | x)p(⋅∣x).

The variational inference problem is now minimizing

D(qλ∥p(⋅∣x))=Ez∼qλ[log⁡qλ(z)−log⁡p(z∣x)]D(q_\lambda\Vert p(\cdot | x)) = \mathbb{E}_{z \sim q_\lambda}\left[\log q_\lambda(z) - \log p(z|x)\right]D(qλ​∥p(⋅∣x))=Ez∼qλ​​[logqλ​(z)−logp(z∣x)]

This still contains the p(z∣x)p(z|x)p(z∣x) term that we can’t evaluate. But we can get rid of that by writing

Ez∼qλ[log⁡qλ(z)−log⁡p(z∣x)]=Ez∼qλ[log⁡qλ(z)−log⁡p(x,z)]+log⁡p(x)\mathbb{E}_{z \sim q_\lambda}\left[\log q_\lambda(z) - \log p(z|x)\right] = \mathbb{E}_{z \sim q_\lambda}\left[\log q_\lambda(z) - \log p(x, z)\right] + \log p(x)Ez∼qλ​​[logqλ​(z)−logp(z∣x)]=Ez∼qλ​​[logqλ​(z)−logp(x,z)]+logp(x)

We have reintroduced log⁡p(x)\log p(x)logp(x), which is intractable, but crucially, it doesn’t depend on λ\lambdaλ. So to solve the minimization problem above, we can also minimize the expected value on the right. Usually, we instead maximize the negative of that:

arg max⁡λEz∼qλ[log⁡p(x,z)−log⁡qλ(z)]\argmax_\lambda \E_{z \sim q_\lambda}\left[\log p(x, z) - \log q_\lambda(z)\right]λargmax​Ez∼qλ​​[logp(x,z)−logqλ​(z)]

The objective

Ez∼qλ[log⁡p(x,z)−log⁡qλ(z)]\E_{z \sim q_\lambda}\left[\log p(x, z) - \log q_\lambda(z)\right]Ez∼qλ​​[logp(x,z)−logqλ​(z)]

is called the evidence lower bound (ELBO) because it is a lower bound on the log evidence log⁡p(x)\log p(x)logp(x):

Ez∼qλ[log⁡p(x,z)−log⁡qλ(z)]=log⁡p(x)−D(qλ∥p(⋅∣x))≤log⁡p(x)\E_{z \sim q_\lambda}\left[\log p(x, z) - \log q_\lambda(z)\right] = \log p(x) - D(q_\lambda\Vert p(\cdot|x)) \leq \log p(x)Ez∼qλ​​[logp(x,z)−logqλ​(z)]=logp(x)−D(qλ​∥p(⋅∣x))≤logp(x)

For now this fact isn’t really interesting, but it will become relevant later.

Maximizing the ELBO is finally a tractable problem: we can write

log⁡p(x,z)=log⁡p(z)+log⁡pθ(x∣z)\log p(x, z) = \log p(z) + \log p_\theta(x|z)logp(x,z)=logp(z)+logpθ​(x∣z)

which is something we can easily evaluate. The expectation is over qλq_\lambdaqλ​ which also doesn’t pose a problem4.

Combining the optimization problems

Let’s briefly recap our progress so far. We originally wanted to find

arg max⁡θlog⁡pθ(x)\argmax_\theta \log p_\theta(x)θargmax​logpθ​(x)

which we rewrote as

arg max⁡θlog⁡pθ(x,z)−log⁡pθ(z∣x)\argmax_\theta \log p_\theta(x, z) - \log p_\theta(z|x)θargmax​logpθ​(x,z)−logpθ​(z∣x)

for an arbitrarily chosen zzz. We then used variational inference to approximate the intractable term as

log⁡pθ(z∣x)≈log⁡qλ∗(z)\log p_\theta(z|x) \approx \log q_{\lambda^*}(z)logpθ​(z∣x)≈logqλ∗​(z)

where λ∗\lambda^*λ∗ is the solution to the variational problem:

λ∗(θ)=arg max⁡λEz∼qλ[log⁡pθ(x,z)−log⁡qλ(z)]\lambda^*(\theta) = \argmax_\lambda \E_{z \sim q_\lambda}\left[\log p_\theta(x, z) - \log q_\lambda(z)\right]λ∗(θ)=λargmax​Ez∼qλ​​[logpθ​(x,z)−logqλ​(z)]

So we could now in principle plug in this approximation and solve

arg max⁡θlog⁡pθ(x)≈log⁡pθ(x,z)−log⁡qλ∗(z)\argmax_\theta \log p_\theta(x) \approx \log p_\theta(x, z) - \log q_{\lambda^*}(z)θargmax​logpθ​(x)≈logpθ​(x,z)−logqλ∗​(z)

but there are problems with this. First, note that λ∗\lambda^*λ∗ depends on θ\thetaθ. If we for example use gradient ascent to optimize over θ\thetaθ, we would need to find the new optimal λ\lambdaλ after each gradient step. Second, using an arbitrarily chosen zzz is kind of silly: we optimized qλq_\lambdaqλ​ such that the entire distribution approximates p(z∣x)p(z|x)p(z∣x) well, we should make use of this entire distribution.

So let’s go back. We know that the solution to

arg max⁡θlog⁡pθ(x)=arg max⁡θlog⁡pθ(x,z)−log⁡pθ(z∣x)\argmax_\theta \log p_\theta(x) = \argmax_\theta \log p_\theta(x, z) - \log p_\theta(z|x)θargmax​logpθ​(x)=θargmax​logpθ​(x,z)−logpθ​(z∣x)

is the same for any zzz. So we can also maximize

Ez∼q[log⁡pθ(x,z)−log⁡pθ(z∣x)]\E_{z \sim q}\left[\log p_\theta(x, z) - \log p_\theta(z|x)\right]Ez∼q​[logpθ​(x,z)−logpθ​(z∣x)]

instead, for an arbitrary distribution qqq. Plugging in our approximation, we get

Ez∼q[log⁡pθ(x,z)−log⁡qλ∗(z)]\E_{z \sim q}\left[\log p_\theta(x, z) - \log q_{\lambda^*}(z)\right]Ez∼q​[logpθ​(x,z)−logqλ∗​(z)]

The question now is which distribution qqq to use. But note that by using q=qλ∗q = q_{\lambda^*}q=qλ∗​, we again get the ELBO, this time as the objective for our original optimization problem. This is a good choice for two reasons:

  1. The ELBO is a lower bound on the evidence, ELBO≤log⁡p(x)\text{ELBO} \leq \log p(x)ELBO≤logp(x). If we used another distribution qqq, we wouldn’t have any guarantee that we’re optimizing for the right thing if the approximation p(z∣x)≈qλ∗(z)p(z|x) \approx q_{\lambda^*}(z)p(z∣x)≈qλ∗​(z) became bad enough. This way, we’re at least optimizing a lower bound on what we really care about.
  2. We saw above that we need to find the new λ∗(θ)\lambda^*(\theta)λ∗(θ) after each update to θ\thetaθ, which is very inefficient. But the ELBO is already our objective for λ\lambdaλ, so now we have the same optimization objective for both parameters and can optimize them jointly.

With this choice of q=qλ∗q = q_{\lambda^*}q=qλ∗​, the joint optimization problem becomes

arg max⁡θ,λEz∼qλ[log⁡pθ(x,z)−log⁡qλ(z)]\argmax_{\theta, \lambda} \E_{z \sim q_\lambda}\left[\log p_\theta(x, z) - \log q_\lambda(z)\right]θ,λargmax​Ez∼qλ​​[logpθ​(x,z)−logqλ​(z)]

We can use the reparameterization trick / pathwise gradients to optimize this efficiently with gradient ascent.

Using an encoder

For a single datapoint xxx, we now have an efficiently solvable problem. But now we get back to the more interesting setting of an entire dataset D\mathcal{D}D. We then want to optimize the likelihood of the entire dataset:

∑x∈Dlog⁡pθ(x)\sum_{x \in \mathcal{D}} \log p_\theta(x)x∈D∑​logpθ​(x)

The problem is that qλ(z)q_\lambda(z)qλ​(z) is supposed to approximate p(z∣x)p(z|x)p(z∣x), so the optimal λ\lambdaλ is different for each datapoint xxx. Full variational inference would mean using a separate parameter λ\lambdaλ for each datapoint. The optimization would then be over θ,λ1,…,λn\theta, \lambda_1, \ldots, \lambda_nθ,λ1​,…,λn​, where λi\lambda_iλi​ is the parameter for the iii-th datapoint. This again gets us into the realm of a huge number of parameters and computational infeasibility.

So instead, we use amortized variational inference. This means that instead of optimizing parameters for qλq_\lambdaqλ​ for each xxx, we learn a function x↦λx \mapsto \lambdax↦λ. This function is trained to approximate the optimal solution λ∗(x)\lambda^*(x)λ∗(x). The downside is that we’re introducing yet another approximation, which can only worsen how well we maximize the likelihood p(x)p(x)p(x). But the big advantage is that evaluating it is much cheaper than solving an entire optimization problem.

In practice, this means we train a neural network fφ(x)f_\varphi(x)fφ​(x) to find the best λ\lambdaλ (in terms of the objective above) for a given xxx. We then use qfφ(x)(z)q_{f_\varphi(x)}(z)qfφ​(x)​(z) in place of qλ∗(z)q_{\lambda^*}(z)qλ∗​(z). To make the notation a bit nicer, we write this as

qφ(z∣x):=qfφ(x)(z)q_\varphi(z|x) := q_{f_\varphi(x)}(z)qφ​(z∣x):=qfφ​(x)​(z)

Then we finally get the VAE objective:

arg max⁡θ,φEz∼qφ[log⁡p(z)pθ(x∣z)−log⁡qφ(z∣x)]\argmax_{\theta, \varphi} \E_{z \sim q_\varphi}\left[\log p(z)p_\theta(x|z) - \log q_\varphi(z|x)\right]θ,φargmax​Ez∼qφ​​[logp(z)pθ​(x∣z)−logqφ​(z∣x)]

Connection to VAEs in practice

As you’ve probably guessed by now, pθ(x∣z)p_\theta(x|z)pθ​(x∣z) is the decoder of a VAE and qφ(z∣x)q_\varphi(z|x)qφ​(z∣x) is the encoder. The ELBO can be rewritten as

Ez∼q(⋅∣x)[log⁡p(z)p(x∣z)−log⁡q(z∣x)]=Ez∼q(⋅∣x)log⁡p(x∣z)−Ez∼q(⋅∣x)log⁡q(z∣x)p(z)=Ez∼q(⋅∣x)log⁡p(x∣z)−D(q(⋅∣x)∥p)\begin{aligned}\E_{z \sim q(\cdot | x)} \left[\log p(z)p(x|z) - \log q(z|x)\right] &= \E_{z \sim q(\cdot | x)} \log p(x|z) - \E_{z \sim q(\cdot | x)} \log \frac{q(z|x)}{p(z)}\\\\\\ &= \E_{z \sim q(\cdot | x)}\log p(x | z) - D(q(\cdot|x)\Vert p)\end{aligned}Ez∼q(⋅∣x)​[logp(z)p(x∣z)−logq(z∣x)]​=Ez∼q(⋅∣x)​logp(x∣z)−Ez∼q(⋅∣x)​logp(z)q(z∣x)​=Ez∼q(⋅∣x)​logp(x∣z)−D(q(⋅∣x)∥p)​

which gives us the interpretation as “reconstruction + regularization loss” that you may have encountered elsewhere (to treat this as a loss that is minimized, you would multiply everything by −1-1−1).

qφq_\varphiqφ​ is typically chosen as a normal distribution, because that makes the KL divergence in the ELBO easy to calculate if p(z)p(z)p(z) is chosen as a unit normal. The choice of pθp_\thetapθ​ depends on the type of data. As mentioned, for binary images we might use independent Bernoulli distributions for each pixel. For continuous output, a normal distribution is a common choice.

Conclusion

We saw how to arrive at VAEs starting from a purely generative motivation, without assuming an autoencoder architecture a priori. Interestingly, this gives a very different impression than the “autoencoder perspective”: what we really care about is the decoder, whereas the encoder is just a useful trick to be able to train the decoder efficiently.

This doesn’t mean that the autoencoder perspective is wrong of course. Having an encoder can be intrinsically useful for some applications, and this is something which is missing in this post. But I think the perspective we took here demonstrates that the VAE architecture is far less arbitrary than it may seem when starting from autoencoders.
